# GDD: AtomicConnection

## 1. Концепция

- **Жанр**: Survival Tower Defense
- **Сеттинг**: Микро-вселенная, молекулы и атомы
- **Целевая аудитория**: Casual / mid-core игроки, которым нравятся компактные survival/TD-игры с ручным управлением под давлением
- **Краткое описание**:
  Игрок кликает на ядро, чтобы заполнять прогресс генерации атома. Когда прогресс заполнен, рядом с ядром появляется атом. Атомы нужно быстро перетаскивать в боевую молекулу, пока враги наступают на ядро. Когда боевая молекула полностью заряжается, игрок тянет направление выстрела и отпускает, чтобы атаковать. Цель — пережить нарастающее давление, дойти до босса и победить его. Между забегами игрок тратит добытые ДНК и изотопы на усиления.
- **Баланс**: конкретные числовые значения вынесены в [AtomicConnection_BALANCE.md](AtomicConnection_BALANCE.md).
- **Пометка `*Если успеем*`**: опциональные идеи для джема. Они не входят в обязательный MVP и берутся только после готовности базового игрового цикла.

---

## 2. Core Game Loop

```
[Клики по ядру] → [Заполнение прогресса генерации]
         ↓
[Прогресс заполнен → спавн атома]
         ↓
[Перетаскивание атома в боевую молекулу]
         ↓
[Заряд растёт до максимума]
         ↓
[Игрок тянет направление выстрела]
         ↓
[Отпускание → атака]
         ↓
[Урон + смерть врага + дроп ДНК]
         ↓
[Threat Level растёт, враги усиливаются]
         ↓
[Босс побеждён → ПОБЕДА / Ядро уничтожено → ПОРАЖЕНИЕ]
         ↓
[Экран прокачки: потратить ДНК/изотопы на улучшения]
         ↓
[Новый забег]
```

## 3. Системы игры

### 3.1 Ядро

- Расположение: центр сцены (фиксированная позиция)
- **Функция**: генерация атомов через прогресс-бар по кликам
- **Генерация**: каждый клик заполняет прогресс генерации; когда шкала заполнена, создаётся атом. Требуемый прогресс и количество атомов при полном заполнении задаются балансом
- **HP**: имеет запас здоровья; базовое значение задаётся балансом
  - Если HP ≤ 0 → **поражение**
- **Апгрейды**:
  - Мульти-атомы при заполнении прогресса генерации
  - Авто-генерация (самостоятельно заполняет прогресс генерации)
  - Увеличение HP
  - Регенерация HP

### 3.2 Атомы

- Создаются у ядра и кружат по его кромке, пока не используются
- **Поведение**: сразу после спавна атом выходит на орбиту вокруг ядра
- **Взаимодействие**: игрок может схватить атом (drag) и перетащить в боевую молекулу
- Если после перетаскивания атом не был прикреплён к боевой молекуле, он возвращается на орбиту ядра
- **Максимум атомов на поле**: не ограничен
- **Время жизни**: бесконечное, пока не использован или не уничтожен при смерти ядра

### 3.3 Боевые молекулы

- Располагаются рядом с ядром
- **Заряд**: шкала заполняется доставленными атомами; требуемое количество задаётся балансом
- Доставленные атомы кружат по орбите боевой молекулы, пока удерживаются в ней
- **Действие при полном заряде**:
  1. Боевая молекула переходит в состояние готовности
  2. Игрок тянет от молекулы в нужную сторону, задавая направление атаки
  3. При отпускании молекула стреляет по выбранному направлению
  4. Тип молекулы определяет форму атаки: одиночный выстрел, массовая атака или другой оружейный паттерн
  5. Заряд сбрасывается
  6. Небольшая задержка (cooldown) перед новым приёмом атомов
- **Урон**: базовое значение задаётся балансом, растёт с апгрейдами
- **Радиус атаки**: базовое значение задаётся балансом
- **Роль**: каждая боевая молекула — отдельный инструмент против своего типа угрозы. Между ними должна читаться простая логика "камень-ножницы-бумага": одна лучше против одиночных жирных целей, другая против толпы, третья закрывает отдельный тип давления.
- **Апгрейды**:
  - Увеличение урона
  - Увеличение радиуса
  - Скорость перезарядки
  - Новые типы боевых молекул
  - Авто-забор атомов (атомы сами летят к выбранной боевой молекуле)

#### 3.3.1 Типы боевых молекул

| Тип | Роль | Хороша против |
|---|---|---|
| Сингл-таргет | Стартовая боевая молекула, стреляет по выбранному направлению в одну цель | Элитки и боссы |
| Массовая | Разблокируемая боевая молекула, атакует область или группу по выбранному направлению | Толпы слабых врагов |
| Третья молекула | Открытый слот для следующего оружейного архетипа | Отдельный тип давления, который не закрывают сингл-таргет и массовая |

- Каждый тип боевой молекулы имеет собственный заряд.
- Заряженная боевая молекула не стреляет сама; она ждёт ручного прицеливания игрока.
- Если заряжено несколько боевых молекул, игрок выбирает нужную, начиная прицеливание от неё.
- Игрок усиливает не только общую мощность, но и набор доступных ответов на разные угрозы.
- Третья молекула пока не зафиксирована; её роль должна закрывать угрозу, которую плохо решают сингл-таргет и массовая молекула.

### 3.4 Враги

#### 3.4.1 Типы врагов

| Тип | Роль |
|---|---|
| Стандартный | Базовая угроза |
| Массовый | Слабый враг, опасен количеством |
| Элитка | Враг с высоким HP |
| Босс | Крупная одиночная угроза |

- При смерти с врага выпадает ДНК

#### 3.4.2 Поведение

- Спавнятся за краем экрана (с разных сторон)
- Двигаются по прямой к ядру
- При достижении ядра: наносят урон (разовый) и исчезают
- *Если успеем*: добавить паттерны движения врагов, например движение по спирали, дуге или с обходом вокруг ядра

#### 3.4.3 Структура уровня и спавн

Текущий MVP: уровень ведёт игрока по таймлайну волн до появления босса.
Победа засчитывается после убийства босса. При завершении игрок получает
настроенную награду изотопами и видит экран прохождения уровня.

Уровень — один полноценный забег. Игрок начинает с базовым набором угроз, проходит через нарастающие фазы и в конце должен победить босса.

| Фаза | Роль |
|---|---|
| Разогрев | Стандартные враги, игрок входит в ритм генерации атомов, зарядки и ручного выстрела |
| Давление массой | Появляются массовые враги, которые проверяют способность быстро чистить толпу |
| Проверка урона | Появляются элитки, которые проверяют сингл-таргет урон |
| Смешанная фаза | Несколько типов врагов идут вместе, заставляя выбирать подходящую боевую молекулу |
| Босс | Финальная одиночная угроза уровня |

- Внутри уровня растёт **Threat Level**.
- Threat Level повышает частоту спавна, открывает новые типы врагов по фазам и усиливает давление на игрока.
- Победа над боссом завершает уровень.
- Столкновение босса с ядром мгновенно проигрывает забег.
- Первое прохождение уровня выдаёт изотопы и открывает следующий уровень.
- Повторные прохождения уровня нужны для фарма ДНК и добора улучшений.
- Конкретный состав волн, частота появления, усиление врагов и момент выхода босса задаются балансом.
- *Если успеем*: добавить replay/prestige-модификаторы для пройденных уровней.

## 4. Мета-прогрессия (Skill Tree)

### 4.1 Валюта

- **ДНК / нуклеотиды**: базовая валюта, получается за уничтожение врагов и тратится на большинство улучшений
- **Изотопы**: редкая валюта, выдаётся только за первое успешное прохождение уровня; используется для ключевых разблокировок
- ДНК и изотопы сохраняются между забегами
- Текущие накопления показываются в HUD забега и на экране прокачки одной
  компактной вью: две строки, сверху ДНК, снизу изотопы, с пиксельными иконками
  валют вместо текстовых названий.
- При поражении игрок сохраняет ДНК, полученную за убийства в текущем забеге
- При победе игрок сохраняет ДНК за убийства; если уровень пройден впервые, дополнительно получает изотопы

### 4.2 Дерево улучшений

```
                    ┌─ Авто-генерация атомов
                    │
    ┌─ Производство ── Авто-мульти-атом
    │               │
    │               └─ Скорость генерации
    │
    │               ┌─ Авто-перенос в выбранную боевую молекулу
    │               │
    ├─ Автоматизация ── Приоритеты авто-переноса
    │               │
    │               └─ Ускорение атомов
    │
────┼─ Боевые улучшения ── Урон сингл-таргет молекулы
    │               │
    │               ├─ Радиус атаки
    │               │
    │               ├─ Скорость перезарядки
    │               │
    │               └─ Массовая молекула (unlock)
    │                              │
    │                              └─ Урон массовой молекулы
    │                              │
    │                              └─ Третья молекула (unlock)
    │
    │               ┌─ HP ядра
    │               │
    └─ Защита ────── Регенерация HP
                    │
                    └─ Щит (каждый N-ный удар блокируется)
```

- Количество уровней и цены улучшений задаются балансом
- Можно обнулить забег и перераспределить?

## 5. Win / Lose условия

| Условие           | Результат  | Действие                     |
|--------------------|------------|------------------------------|
| Босс побеждён | **Победа** | ДНК за убийства сохраняется; за первое прохождение уровня выдаются изотопы и открывается следующий уровень |
| Ядро уничтожено | **Поражение** | ДНК за убийства сохраняется |
| Игрок в любой момент может выйти | Прогресс сохранён | Возврат в меню |

## 6. UI / UX

### 6.1 HUD (игровой экран)

- **Верхний бар**:
  - Таймер обратного отсчёта (MM:SS)
  - Золото
- **Ядро**:
  - HP
  - Прогресс-бар генерации атома
- **Боевые молекулы**:
  - Индикатор заряда
  - Состояние готовности к выстрелу
- **Подсказки**: туториал в начале

### 6.2 Экран победы/поражения

- Текущий MVP-экран прохождения открывается после убийства босса,
  показывает награду изотопами и предлагает "Restart" или "Main Menu".
- Текст "ПОБЕДА" / "ПОРАЖЕНИЕ"
- Статистика забега (убито врагов, получено ДНК, получены изотопы при первом прохождении)
- Кнопка "Пройти прокачку" → дерево апгрейдов
- Кнопка "Начать заново"

### 6.3 Экран прокачки (Skill Tree)

- Дерево улучшений
- Доступные / купленные ячейки
- Текущие ДНК и изотопы
- Кнопка "Начать забег"
- В текущем тестовом MVP дерево талантов открывается напрямую из главного меню
  через кнопку "Update", без промежуточного экрана результата. Позже этот вход
  должен перейти на экран победы/поражения.
- Визуально дерево талантов строится как граф: стартовая центральная ячейка
  открывает зависимые ветки, линии показывают требования между талантами.
- Граф можно приближать и отдалять колесом мыши. Описание таланта показывается
  в tooltip при наведении на ячейку, чтобы дерево оставалось читаемым на любом
  масштабе.
- Текущий набор талантов для MVP: HP ядра, скорость генерации атомов, скорость
  генерации снарядов, урон боевой молекулы, радиус атаки, авто-генерация атомов
  и авто-сбор атомов в боевую молекулу.
- На текущем этапе применяются эффекты HP ядра, генерации атомов, генерации
  снарядов и урона. Радиус атаки зарезервирован для будущей автострельбы.
  Авто-генерация и авто-сбор остаются разблокировками для следующего
  gameplay-среза.

### 6.4 Главное меню

- "Играть"
- "Update" для открытия дерева талантов
- "Reset" для удаления сохранённых данных
- Настройки (звук, громкость)
- В нижнем левом углу отображается выбор открытого уровня: `Level N`.
  Стрелка влево показывается, если доступен предыдущий уровень, стрелка вправо —
  если выбран не самый новый открытый уровень. После первого прохождения текущего
  максимального уровня открывается следующий уровень и он становится выбранным.

## 7. Этапы MVP (Minimum Viable Product)

| Этап | Что входит |
|---|---|
| **MVP-1** | Ядро + прогресс генерации атома по кликам + атомы + drag & drop в боевую молекулу + ручной выстрел + стандартный враг + босс |
| **MVP-2** | Четыре типа врагов + структура уровня + Threat Level + система урона + HP ядра + экран победы/поражения |
| **MVP-3** | Дерево прокачки + сохранение + валюты |
| **MVP-4** | Массовая молекула + третий оружейный архетип + авто-апгрейды + баланс |
| **Полишинг** | Туториал, меню, финальная доводка ощущений |
