# GDD: AtomicConnection

## 1. Концепция

- **Жанр**: Survival Tower Defense
- **Сеттинг**: Микро-вселенная, молекулы и атомы
- **Вид**: Top-down 2D (вертикальная камера)
- **Платформы**: Web (WebGL) / PC Standalone
- **Целевая аудитория**: Casual / mid-core игроки (любители TD и merge-механик)
- **Краткое описание**:
  Игрок кликает на главную молекулу, чтобы генерировать атомы. Атомы нужно перетаскивать в боевую молекулу. Когда она полностью заряжается — авто-выстрел по врагам. Надо выжить до конца таймера. Между забегами — дерево прокачки для усиления.
- **Баланс**: конкретные числовые значения вынесены в [AtomicConnection_BALANCE.md](AtomicConnection_BALANCE.md).

---

## 2. Core Game Loop

```
[Клик по главной молекуле] → [Спавн атома]
         ↓
[Перетаскивание атома в боевую молекулу]
         ↓
[Заряд растёт до максимума]
         ↓
[Авто-выстрел по ближайшему врагу]
         ↓
[Урон + смерть врага + дроп золота]
         ↓
[Волны врагов нарастают по таймеру]
         ↓
[Таймер истёк → ПОБЕДА / Смерть → ПОРАЖЕНИЕ]
         ↓
[Экран прокачки: потратить золото на улучшения]
         ↓
[Новый забег]
```

## 3. Системы игры

### 3.1 Главная молекула (Центральное ядро)

- Расположение: центр сцены (фиксированная позиция)
- **Функция**: генерация атомов по клику
- **Генерация**: создаёт атомы по клику; базовое количество задаётся балансом
- **HP**: имеет запас здоровья; базовое значение задаётся балансом
  - Если HP ≤ 0 → **поражение**
- **Визуал**: пульсирующая сфера, меняет цвет при низком HP
- **Апгрейды**:
  - Мульти-атомы за клик
  - Авто-генерация (каждые N секунд спавнит атом)
  - Увеличение HP
  - Регенерация HP

### 3.2 Атомы

- Создаются у главной молекулы
- **Поведение**: сразу после спавна — физичное движение (имитация вылета из молекулы)
- **Взаимодействие**: игрок может схватить атом (drag) и перетащить в боевую молекулу
- **Максимум атомов на поле**: не ограничен (но пул объектов для оптимизации)
- **Время жизни**: бесконечное, пока не использован или не уничтожен при смерти молекулы
- **Анимация**: вращение, свечение

### 3.3 Боевая молекула

- Располагается рядом с главной (настраиваемая позиция)
- **Заряд**: шкала заполняется доставленными атомами; требуемое количество задаётся балансом
- **Действие при полном заряде**:
  1. Авто-прицеливание: найти ближайшего врага в радиусе
  2. Выстрел: снаряд летит во врага
  3. Заряд сбрасывается
  4. Небольшая задержка (cooldown) перед новым приёмом атомов
- **Урон**: базовое значение задаётся балансом, растёт с апгрейдами
- **Радиус атаки**: базовое значение задаётся балансом
- **Апгрейды**:
  - Увеличение урона
  - Увеличение радиуса
  - Скорость перезарядки
  - Вторая боевая молекула (AoE-урон по площади)
  - Авто-забор атомов (атомы сами летят к боевой молекуле)

**Вторая (AoE) молекула**:
- Разблокируется в дереве прокачки
- Атакует по площади (радиусный взрыв)
- Имеет отдельный заряд, но медленнее стреляет
- Визуальный стиль: отличается цветом/размером

### 3.4 Враги

#### 3.4.1 Типы врагов

| Тип    | Роль | Визуал                |
|--------|------|------------------------|
| Базовый| Стандартный враг | Маленький шар          |
| Быстрый| Быстрое давление | Маленький, вытянутый   |
| Толстый| Живучий враг | Большой, тяжёлый       |

- При смерти с врага выпадает золото (монета, летит в счётчик)

#### 3.4.2 Поведение

- Спавнятся за краем экрана (с разных сторон)
- Двигаются по прямой к главной молекуле
- При достижении главной молекулы: наносят урон (разовый) и исчезают
- Визуальный фидбек: мигание при получении урона, Particle Effect при смерти

#### 3.4.3 Спавн (Волны)

Система прогрессии сложности в рамках одного забега:

| Фаза | Событие                        |
|------|--------------------------------|
| Старт | Первые враги                  |
| Раннее усиление | Учащение спавна       |
| Средняя фаза | Добавляются быстрые враги |
| Поздняя фаза | Добавляются толстые враги |
| Предфинал | Мини-босс                    |
| Финал | Финальная волна                 |

- Параметры управляются из конфига (ScriptableObject / JSON)

## 4. Мета-прогрессия (Skill Tree)

### 4.1 Валюта

- **Золото**: выпадает с убитых врагов, тратится на улучшения
- Сохраняется между забегами (PlayerPrefs или JSON)

### 4.2 Дерево улучшений

```
                    ┌─ Авто-генерация атомов
                    │
    ┌─ Производство ── Авто-мульти-атом
    │               │
    │               └─ Скорость генерации
    │
    │               ┌─ Авто-перенос в боевую молекулу
    │               │
    ├─ Автоматизация ── Авто-перенос в AoE молекулу
    │               │
    │               └─ Ускорение атомов
    │
────┼─ Боевые улучшения ── Урон
    │               │
    │               ├─ Радиус атаки
    │               │
    │               ├─ Скорость перезарядки
    │               │
    │               └─ AoE молекула (unlock)
    │                              │
    │                              └─ Урон AoE
    │
    │               ┌─ HP главной молекулы
    │               │
    └─ Защита ────── Регенерация HP
                    │
                    └─ Щит (каждый N-ный удар блокируется)
```

- Количество уровней и цены улучшений задаются балансом
- Можно обнулить забег и перераспределить?

## 5. Win / Lose условия

| Условие           | Результат  | Действие                     |
|--------------------|------------|------------------------------|
| Таймер истёк | **Победа** | Награда + бонусное золото    |
| Главная молекула уничтожена | **Поражение** | Золото сохраняется (частично?) |
| Игрок в любой момент может выйти | Прогресс сохранён | Возврат в меню |

## 6. UI / UX

### 6.1 HUD (игровой экран)

- **Верхний бар**:
  - Таймер обратного отсчёта (MM:SS)
  - Золото (счётчик, с анимацией получения)
- **Главная молекула**:
  - HP bar (полоска над молекулой)
  - Индикатор генерации (если стоит авто-апгрейд)
- **Боевая молекула**:
  - Индикатор заряда (сегменты заполняются атомами)
- **Подсказки**: туториал в начале (стрелки, текст)

### 6.2 Экран победы/поражения

- Затенение сцены
- Текст "ПОБЕДА" / "ПОРАЖЕНИЕ"
- Статистика забега (убито врагов, получено золота)
- Кнопка "Пройти прокачку" → дерево апгрейдов
- Кнопка "Начать заново"

### 6.3 Экран прокачки (Skill Tree)

- Визуальное дерево с иконками и связями
- Доступные / купленные ячейки
- Текущее золото
- Кнопка "Начать забег"

### 6.4 Главное меню

- "Играть"
- "Прокачка"
- Сеттинг (звук, громкость)

## 7. Арт / Визуальная концепция

### 7.1 Стилистика

- Минимализм, яркие цвета, научная тематика
- Neon/Glow эффекты на молекулах и атомах
- Тёмный фон (космос / микро-среда)

### 7.2 Палитра

- Главная молекула: голубой/циан (ядро)
- Боевая молекула: оранжевый/красный
- AoE молекула: фиолетовый
- Атомы: белые/жёлтые, с лёгким свечением
- Враги: зелёные (базовый), жёлтые (быстрый), красные (толстый)
- Фон: тёмно-фиолетовый/чёрный с частицами

### 7.3 Разрешение

- Canvas: Full HD (1920×1080) или адаптивное под экран

## 8. Технические требования

- **Unity**: 2022.3 LTS (LTS версия)
- **Render Pipeline**: URP или Built-in (урок — URP для 2D света)
- **Input**: New Input System (Mouse + клавиатура)
- **Target**: WebGL (Desktop) + Standalone (Windows/Mac)
- **Архитектура**:
  - ScriptableObjects для конфигов (враги, волны, апгрейды)
  - Object Pooling для атомов, врагов, снарядов
  - Сохранение: JSON (Newtonsoft.Json или System.Text.Json)
- **Оптимизация**: Object pooling, LOD для атомов, отключение физики для статичных объектов

## 9. Этапы MVP (Minimum Viable Product)

| Этап | Что входит | Кому |
|------|-----------|------|
| **MVP-1** | Главная молекула + клик-генерация + атомы + drag & drop в боевую молекулу + выстрел + один тип врага + таймер | Разработчик |
| **MVP-2** | Три типа врагов + волны + система урона + HP главной молекулы + экран победы/поражения | Разработчик |
| **MVP-3** | Дерево прокачки + сохранение + золото | Разработчик |
| **MVP-4** | Арт, анимации, VFX, звук | Художник |
| **MVP-5** | AoE молекула + авто-апгрейды + баланс | Разработчик + Художник |
| **Полишинг** | UI-анимации, туториал, меню, оптимизация | Оба |
